hexo写文章的注意事项

发表于 2020-11-29 分类于 hexo ， 前端 Valine：
本文字数： 456 阅读时长 ≈ 1 分钟

生成文件方式

在hexo的根目录使用命令hexo new "文件名，也是文章标题"，会自动生成带有Front-matter的md文件，推荐使用。

其次，也可以直接在hexo的根目录下的source文件夹下的_post文件夹中新建md文件，然后自己按Front-matter的格式定义文章信息。

存放文件夹

使用由于next主题中也有一个source文件夹，所以有时候会跑到主题的文件夹中去，但只要记得文章就是存放在_post文件夹下，另外，文章的层级也不属于主题，还是很容易区分的。

文章字号

在typora编辑器中，文章内容的小标题采用四级标题比较合适，这样文章内容的标题在hexo上显示就不会超过文章的标题，造成头轻脚重的现象，便于阅读。

Front-matter信息格式问题

在Front-matter中定义tags和categories的信息时，注意格式问题。

- hexo

  • 具体定义的信息需要换行
  • “-”的左右必须各有一个空格，否则在hexo g生成文件时会报错
  • 在Front-matter中是可以使用#来写注释的

标签、分类的区别以及Front-matter的使用

发表于 2020-11-29 分类于 hexo ， 前端 Valine：
本文字数： 311 阅读时长 ≈ 1 分钟

标签、分类的区别

只有文章支持分类和标签，您可以在 Front-matter 中设置。在其他系统中，分类和标签听起来很接近，但是在 Hexo 中两者有着明显的差别：分类具有顺序性和层次性，也就是说 Foo, Bar 不等于 Bar, Foo；而标签没有顺序和层次。

Front-matter的使用

Front-matter 是文件最上方以 --- 分隔的区域，用于指定文件信息的变量，以下是预先定义的参数：

参数描述默认值
layout布局
title标题
date建立日期文件建立日期
updated更新日期文件更新日期
comments开启文章的评论功能true
tags标签（不适用于分页）
categories分类（不适用于分页）
permalink覆盖文章网址

hexo常用命令

发表于 2020-10-02 分类于 hexo ， 前端 Valine：
本文字数： 575 阅读时长 ≈ 1 分钟

hexo clean

hexo clean

  • 清除缓存文件 db.json 和已生成的静态文件 public
  • 网站显示异常时可以执行这条命令试试

hexo g

hexo g

  • 生成网站静态文件到默认设置的 public 文件夹

  • 便于查看网站生成的静态文件或者手动部署网站

  • 如果使用自动部署，不需要先执行该命令

  • hexo g 是 hexo generate 的缩写，命令效果一致

hexo s

hexo s

  • 启动本地服务器，用于预览主题。默认地址： http://localhost:4000/
  • 要想本地预览效果必须先打开server
  • 预览的同时可以修改文章内容或主题代码，保存后刷新页面即可
  • 对 Hexo 根目录 _config.yml 的修改，需要重启本地服务器后才能预览效果

hexo d

hexo d

  • 自动生成网站静态文件，并部署到设定的仓库

  • hexo d 是 hexo deploy 的缩写，命令效果一致

hexo new

hexo new "学习笔记"

  • 新建一篇标题为”学习笔记”的文章，因为标题里有空格，所以加上了引号

  • 文章标题可以在对应 md 文件里改，新建时标题可以写的简单些

hexo new page

hexo new page aboutme

  • 新建一个标题为 aboutme 的页面，默认链接地址为 主页地址/aboutme/

  • 标题可以为中文，但一般习惯用英文

  • 页面标题和文章一样可以随意修改

  • 页面不会出现在首页文章列表和归档中，也不支持设置分类和标签
